# 📚 O que é IA Generativa?

A Inteligência Artificial Generativa é capaz de produzir textos, códigos, imagens e outros conteúdos a partir de comandos escritos pelo usuário.

Na programação ela pode:

✅ gerar código;

✅ explicar código;

✅ corrigir erros;

✅ documentar projetos;

✅ sugerir melhorias.

> **Importante:** A IA auxilia o desenvolvedor, mas **não substitui** o conhecimento técnico nem a necessidade de testar o código produzido.

---

# 🧠 Módulo 1 — A IA não substitui o programador

A IA pode acelerar o desenvolvimento, porém ela não conhece o contexto completo do seu projeto.

Sempre:

* leia o código gerado;
* teste o funcionamento;
* verifique se atende ao objetivo;
* compreenda antes de utilizar.

# ✍️ Módulo 2 — Anatomia de um Bom Prompt

Observe a diferença.

### ❌ Prompt ruim

```text
Faça um sistema.
```

A IA terá poucas informações para responder.

---

### ✅ Prompt bem elaborado

```text
Crie um jogo em HTML, CSS e JavaScript para estudantes iniciantes.

O projeto deve conter apenas:

- index.html
- style.css
- script.js

Utilize código simples, comentado e fácil de entender.
```

---

## Um bom prompt deve conter

* 🎯 Objetivo;
* 📖 Contexto;
* 💻 Tecnologias utilizadas;
* 📌 Restrições;
* 👨‍🎓 Público-alvo ou nível do aluno;
* 📦 Resultado esperado.

---

# 💬 Módulo 3 — Como conversar com a IA

A utilização da IA é um processo iterativo.

```text
Faça uma pergunta
        ↓
Receba a resposta
        ↓
Teste o código
        ↓
Identifique problemas
        ↓
Peça correções
        ↓
Teste novamente
        ↓
Melhore o projeto
```

Quanto melhor for a conversa, melhores serão os resultados.

---

# ✅ Módulo 4 — Validando as Respostas

Nunca copie um código sem analisá-lo.

Perguntas úteis para fazer à IA:

* Esse código realmente funciona?
* Existem erros?
* Há uma solução mais simples?
* Explique linha por linha.
* Como posso melhorar este código?
* Existe algum problema de segurança?

---

# ⚠️ Módulo 5 — Erros Comuns da IA

Nem toda resposta da IA está correta.

Fique atento quando ela:

* inventa bibliotecas inexistentes;
* utiliza versões antigas de tecnologias;
* cria funções que não existem;
* responde algo diferente do solicitado;
* gera código que não executa.

Por isso, **sempre teste**.
